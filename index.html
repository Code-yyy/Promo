IoMT Shield: Real-time protection for Medical IoT gateways

IoMT Shield is an end-to-end security framework for remote patient monitoring (RPM) environments. It secures telemetry in transit, detects network attacks at the edge using TinyML, gathers threat intelligence via honeypots, and enables automated mitigation through SDN and a dashboard.

What’s inside IoMT Shield

Four integrated components that work together for secure medical IoT operation.

1) Secure Transmission

Patient telemetry is protected before reaching the gateway using lightweight encryption and integrity checks.

ESP32 → Gateway Confidentiality Integrity

2) TinyML Intrusion Detection

A lightweight deep learning model runs on Raspberry Pi to detect attacks in real time and classify traffic.

TinyML Edge inference events.jsonl

3) Honeypot Intelligence

A decoy service attracts attackers, captures scans/brute-force patterns, and generates threat intelligence logs.

Decoy Threat intel Dataset support

4) SDN + Dashboard

Structured security events trigger SDN rules for blocking and provide real-time visibility through a dashboard.

Detect → Block Dashboards Policy automation

How it works (Simple flow)

From device telemetry to automated network response.

Capture & Protect

Medical data is generated and transmitted securely to the gateway with encryption + integrity verification.

Detect at the Edge

The Raspberry Pi analyzes traffic and labels activity as BENIGN / SUSPICIOUS / ATTACK in real time.

Respond & Visualize

Logs feed SDN + dashboard: block malicious sources and display alerts with evidence and metadata.
